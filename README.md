# web-home-archive

## 项目简介

web-home-archive 是一个用于归档和发布多个独立 HTML 页面的仓库。该仓库不针对任何特定域名或网站，仅作为静态页面的存储和展示空间，方便对各类网页内容进行历史记录或临时发布。

## 目录说明

```
web-home-archive/
├── pages/          # 存放归档的 HTML 页面文件
├── assets/         # 页面引用的静态资源（图片、样式、脚本等）
├── index.html      # 仓库首页（可自定义）
└── README.md       # 本说明文件
```

- `pages/` 目录：每个子文件夹或文件对应一个独立的 HTML 页面归档，文件名或文件夹名应能简要标识页面内容。
- `assets/` 目录：存放页面所需的共用或私有资源，建议按页面归类存放。

## 页面归档说明

- 每个归档的 HTML 页面应为一个完整的、可独立运行的文档。
- 页面文件名建议使用英文或拼音，避免特殊字符。
- 如需保留多个版本，可在 `pages/` 下按日期或版本号创建子目录。
- 页面中引用的资源路径请使用相对路径，确保在仓库内可直接访问。

## 维护说明

### 添加新页面

1. 将 HTML 文件放入 `pages/` 目录（或新建子文件夹）。
2. 如有配套资源，放入 `assets/` 对应位置。
3. 更新 `index.html`（可选），添加指向新页面的链接。

### 更新已有页面

- 直接覆盖原文件，或按版本号存放新版本。
- 建议在页面注释中注明最后修改时间。

### 删除页面

- 移除对应文件或文件夹。
- 同步更新 `index.html` 中的链接。

## 使用建议

- 本仓库适合存放临时性、实验性或存档性质的 HTML 页面。
- 请勿存放包含敏感信息、违法内容或侵犯他人权益的页面。
- 如页面数量较多，建议在 `pages/` 内建立分类子目录。
